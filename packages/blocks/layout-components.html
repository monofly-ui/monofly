Overview Three components, one job: spatial structure with no business logic.

The layout layer sits second-from-bottom in the dependency stack — above primitives, below everything else:

primitives → layout → compositions → blocks → templates

All three share a common philosophy that makes them ideal for generative / agentic tooling (Figma Make Kits):

PrincipleHow it shows up in the code
Constrained, enumerated propsSpacing is never a free number — only the token scale 100 200 300 400 600 800 1200 1600 (+ 4000 for Section). An LLM can't emit an off-grid gap="13px".
Token indirectionProps map to classnames → classnames set CSS custom properties → properties read design tokens. Re-theme by swapping tokens, never component code.
Polymorphic & transparentEach spreads ComponentPropsWithoutRef<"div">, so id, aria-*, data-*, onClick all pass straight through to a real DOM node.
Composable pairsFlex/FlexItem and Grid/GridItem — the container declares the system, the item opts into a role within it.

2 · The ratio engine Flex's type + FlexItem's size — a responsive column system with zero JS and zero per-component media queries.

  
This is the most sophisticated part of the layout layer and what makes it genuinely different from a plain flex wrapper. Setting type to half, third or quarter turns the Flex into a fractional column grid whose column counts change at breakpoints — but the breakpoint logic lives entirely in responsive.css as CSS custom properties, not in the component.


  
How it works

  
    
    
  1. responsive.css defines ratio tokens (e.g. --sds-responsive-ratio-column-third-major) and redefines them inside @media blocks at 600px and 1024px.
    2. 
    
  2. The Flex CSS derives --column-count, --base-width and --column-span from those tokens with pure calc(), accounting for the gap.
    3. 
    
  3. Each child's flex-basis/max-width is computed from its size role (major/minor/half/full).
    4. 
  

  
Net effect: a type="third" row is 3-up on desktop, 1-up on mobile, reflowing on resize without React re-rendering or a single component-level @media query.

Why this is clever: responsive behaviour is a data concern (three lines of token overrides) rather than a code concern. A designer can re-tune every breakpoint in the whole system by editing responsive.css — no component touches, no rebuild of logic, and the cascade does the work on the GPU/compositor, not in React.







  
3 · Grid An escape hatch to full CSS Grid power, with the gap API kept on-token.

  
Where Flex hides flexbox behind an opinionated ratio system, Grid is the opposite: it exposes raw grid-template-columns/rows strings for full control, but still routes gaps through the token scale and applies them as inline styles + classnames.

Flex vs Grid — when to reach for which. Use Flex type="third|quarter|half" when you want the design system's built-in responsive reflow for free (cards, plan grids, content+sidebar). Drop to Grid when you need explicit 2-D placement, named areas, spanning, or a template flex can't express — and you'll own the responsive breakpoints yourself (via useMediaQuery swapping the columns string).







  
4 · Section The vertical-rhythm + theming primitive. Every page is a stack of Sections.

  
Section owns three things the other two don't: semantic element choice, page-level vertical padding, and background/theme variants — including an image variant with a built-in scrim. It is the only layout component with a discriminated-union prop type (the image variant requires a src; the others forbid it).


  

    
PropTypeDefaultNotes

    
elementTypesection · header · footersectionRenders the matching semantic tag.

    
variantsubtle · neutral · brand · stroke · imagesubtleimage is a separate union arm that requires src.

srcstring (required iff variant="image")—Background image; rendered via SdsImage with a scrim ::before.

6 · Performance & engineering characteristics

  

    
PropertyDetail

    
No runtime layout JSAll sizing is CSS calc() over custom properties. React only toggles classnames + a couple of inline CSS vars. Resize/reflow is handled by the browser's compositor, never a JS resize listener.

    
Responsive without re-renderThe ratio engine reacts to @media in responsive.css. Rotating a phone or resizing reflows the grid with zero React work — no useState, no useMediaQuery for the column behaviour. (Blocks like PricingGrid still use useMediaQuery, but only for content choices like card size, not layout columns.)

    
Tiny, static class setsclsx assembles a deterministic, bounded set of classnames. No styled-components/emotion runtime, no per-render style injection, no hash churn.

    
Zero runtime dependenciesThe only import is clsx (bundled). Enforced repo-wide by the prebuild guard — the layout layer ships pure CSS + a thin React wrapper inside the single dist/index.js.

    
Tree-shakeableEach component is a named export through the barrel; CSS is marked sideEffects so styles survive shaking while unused JS is dropped.

    
Theme-swappable at runtimeBecause everything reads tokens, dark mode / re-branding is a custom-property swap on :root — no component re-render, no recompile.

    
Accessible & semanticSection emits real <section>/<header>/<footer>; the image variant carries a presentation role + scrim for contrast. All three forward aria-*/role via prop spread.

  

  
Why it suits agentic / generative tooling. The constrained, enumerated prop surface (no arbitrary pixels, a fixed set of variants) means an LLM emitting these components cannot produce off-grid spacing or an undefined variant — the type system and classname map make malformed output unrepresentable. That's the whole thesis of a "Figma Make Kit": maximum expressiveness, minimum degrees of freedom.







  
7 · Cheatsheet

  
// Spacing scale (gap / padding) — the ENTIRE spacing vocabulary
100=4px  200=8px  300=12px  400=16px  600=24px  800=32px  1200=48px  1600=64px   (Section adds 4000=160px)

// Flex — opinionated flexbox + responsive columns
<Flex direction gap alignPrimary alignSecondary wrap container type/>
   type: auto | half | third | quarter        // responsive column model
<FlexItem size="full|major|minor|half|fill" />

// Grid — raw CSS Grid, on-token gaps
<Grid columns rows gap columnGap rowGap flow justifyItems alignItems container />
<GridItem column row area />

// Section — semantic + theming + vertical rhythm
<Section elementType="section|header|footer"
         variant="subtle|neutral|brand|stroke|image"
         padding paddingTop paddingBottom  src(image only) />

// Mental model
Flex   → 1-D flow + free responsive reflow (cards, sidebars, toolbars)
Grid   → 2-D explicit placement (app shells, dashboards, galleries)
Section→ the page's horizontal bands (stack them to build a page)
